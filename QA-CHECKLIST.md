# WBS管理システム QAチェックリスト

## テスト実施前の準備

- [ ] Vercelデプロイが成功している（Deployments画面でReady）
- [ ] Firebase Authが有効（Google認証ON）
- [ ] Firestoreセキュリティルールが最新版
- [ ] Firestoreインデックスが4つ全て「有効」状態
- [ ] テスト用ブランド・テンプレートが登録済み

---

## 1. 認証・ログイン

### 1-1. 初回ログイン（未登録ユーザー）
- [ ] Googleログインボタンが動作する
- [ ] Google認証ポップアップが表示される
- [ ] ログイン後「承認待ち」画面が表示される
- [ ] Firestoreのusersコレクションに `status: "pending"` で作成される
- [ ] 「別のアカウントでログイン」リンクが動作する

### 1-2. 事前登録ユーザーのログイン
- [ ] ユーザー管理で事前登録したメールアドレスでログイン
- [ ] 承認待ちにならず、即座にダッシュボードが表示される
- [ ] 事前登録時に設定したロール・店舗が反映されている
- [ ] preRegisteredUsersのドキュメントが削除されている

### 1-3. 承認フロー
- [ ] admin画面で承認待ちユーザーが表示される
- [ ] 「オーナーとして承認」→ ロールがownerになる
- [ ] 「PMとして承認」→ ロールがpmになる
- [ ] 「拒否」→ ユーザーが削除される
- [ ] 承認後、そのユーザーがリロードするとダッシュボードが表示される

### 1-4. ログアウト
- [ ] ログアウトボタンが動作する
- [ ] ログアウト後、ログイン画面にリダイレクトされる
- [ ] ログアウト後、直接URLアクセスしてもログイン画面に遷移する
